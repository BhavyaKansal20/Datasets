# 🤝 Contributing Guidelines

Thank you for your interest in contributing! 🚀
This repository is maintained by **Bhavya Kansal** and is open to meaningful contributions.

---

## 🧠 Contribution Scope

You can contribute by:

* Adding new datasets
* Improving existing datasets
* Fixing errors or inconsistencies
* Enhancing documentation
* Suggesting better data sources

---

## 🛠 How to Contribute

1. **Fork** the repository
2. Create a new branch:

```bash
git checkout -b feature/your-feature-name
```

3. Make your changes
4. Commit with a clear message:

```bash
git commit -m "feat: add dataset XYZ"
```

5. Push your branch:

```bash
git push origin feature/your-feature-name
```

6. Open a **Pull Request (PR)**

---

## 📦 Dataset Guidelines

* Provide **clean and well-structured data**
* Mention **source of dataset**
* Ensure dataset is **legally shareable**
* Add a short **description in README**
* Avoid uploading extremely large or sensitive data

---

## 🧾 Code & Documentation Standards

* Keep code simple and readable
* Use meaningful names
* Add comments where needed
* Keep changes focused and minimal

---

## 🐛 Reporting Issues

When reporting issues:

* Clearly describe the problem
* Include steps to reproduce
* Add screenshots/logs if needed

---

## 🔀 Pull Request Rules

* Keep PRs small and focused
* Reference related issues (if any)
* Ensure changes are properly tested
* Be respectful during reviews

---

## ⚠️ Important Notes

* Do not upload copyrighted or restricted datasets
* Respect original dataset licenses
* Follow ethical data usage practices
